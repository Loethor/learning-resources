# Contribution Guidelines for GameDev Learning Resources

Thank you for your interest in contributing to the GameDev Learning Resources repository! We welcome contributions from everyone, whether you're a seasoned developer or just starting out. Please follow the guidelines below to ensure a smooth contribution process.

## How to Contribute

1. **Choose a Topic**: Review the existing topics in the `topics` directory. If you have resources related to architecture, Godot, networking, videos, or blogs, you can contribute to the relevant file.

2. **Adding Resources**:
   - For articles, videos, or blogs, please include the title, a brief description, and the link.
   - Ensure that the resources are relevant to the topic and provide value to learners.

3. **Format for Submissions**:
   - Use the following format for each resource:
     ```
     - [Title of the Resource](URL) - Brief description of the resource.
     ```
   - Make sure to maintain consistent formatting throughout the file.

4. **Pull Requests**:
   - Fork the repository and create a new branch for your contribution.
   - After making your changes, submit a pull request with a clear description of what you have added.

5. **Review Process**:
   - Your pull request will be reviewed by the maintainers. They may suggest changes or improvements before merging.

## Code of Conduct

Please adhere to a code of conduct that promotes a welcoming and inclusive environment. Be respectful and constructive in your feedback and interactions.

## Questions?

If you have any questions or need assistance, feel free to open an issue in the repository or reach out to the maintainers.

Thank you for helping to build a valuable resource for game development learners!
